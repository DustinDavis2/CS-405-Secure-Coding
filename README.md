# CS-405 Secure Coding

## Overview

This repository contains my completed coursework and final projects for SNHU's CS-405 Secure Coding course. Throughout this course, I learned how secure coding practices help prevent common software vulnerabilities by identifying risks early in the development process and implementing security throughout the software development lifecycle.

## Course Objectives

The projects in this repository focus on:

- Secure coding standards
- Software vulnerability identification
- Risk assessment and mitigation
- Secure software development practices
- Security policy development
- Zero trust security principles

## Repository Contents

This repository includes the following course deliverables:

- Project One Security Policy
- Project Two Security Presentation
- Portfolio Reflection Journal
- Coding project ZIP files completed throughout the course

## Skills Demonstrated

During this course, I worked with several security concepts and development practices, including:

- Integer overflow and underflow protection
- SQL injection prevention
- Exception handling
- File encryption
- Static code analysis
- Unit testing
- Threat modeling
- Secure coding standards
- Security policy implementation
- Zero trust security

## Key Takeaways

One of the biggest lessons from this course was that security should be built into software from the beginning instead of being added after development is complete. Identifying vulnerabilities early, following secure coding standards, and evaluating security risks throughout development helps produce more reliable and secure applications.
